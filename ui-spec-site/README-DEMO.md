# UI 规范示例站点

基于《前端开发设计规范》创建的完整示例项目,展示了符合 Ant Design 设计体系的各类组件应用。

## 📋 功能特性

本项目包含以下示例页面:

### 1. 首页 (/)
- 展示所有示例页面的导航入口
- 设计价值观卡片展示
- 响应式布局

### 2. 表单示例 (/form-demo)
遵循表单设计规范:
- ✅ 单列布局,视觉流清晰
- ✅ 即时校验,错误提示明确
- ✅ 最小化输入,提供默认值
- ✅ 标签右对齐,表意清晰
- ✅ 合理的按钮位置(主按钮在右)

**功能包括:**
- 用户注册表单
- 多种表单控件(输入框、选择器、日期选择、多选框等)
- 表单验证(自定义验证器)
- 加载状态反馈

### 3. 列表示例 (/list-demo)
遵循数据展示规范:
- ✅ 信息层次清晰(通过字重、颜色区分)
- ✅ 操作位置合理(行操作在行尾,批量操作在表格上方)
- ✅ 分页加载
- ✅ 筛选和排序功能
- ✅ 批量操作支持

**功能包括:**
- 用户列表展示
- 筛选和搜索
- 批量删除
- 分页控制
- 表格固定列

### 4. 弹窗示例 (/dialog-demo)
遵循对话框设计规范:
- ✅ 合理的尺寸(520px/720px/960px)
- ✅ 清晰的操作按钮
- ✅ 足不出户原则(模态操作)
- ✅ 支持多种关闭方式

**功能包括:**
- 基础对话框
- 表单对话框
- 确认对话框
- 全屏对话框
- 四个方向的抽屉
- 嵌套对话框

### 5. 反馈示例 (/feedback-demo)
遵循反馈设计规范:
- ✅ 即时反馈
- ✅ 明确的反馈类型(成功/警告/错误/信息)
- ✅ 合适的位置(消息在顶部,通知在右上角)
- ✅ 适当的停留时间

**功能包括:**
- Message 消息提示
- Notification 通知
- Loading 加载状态
- Progress 进度条
- Result 结果展示
- Skeleton 骨架屏
- Empty 空状态
- Alert 警告提示

## 🎨 设计规范遵循

所有页面严格遵循《前端开发设计规范》:

### 设计价值观
- **自然**: 降低用户认知成本
- **确定性**: 高确定性、低合作熵
- **意义感**: 富有意义感的人机交互
- **生长性**: 人机共同生长

### 全局样式
- **色彩系统**: 使用 Element Plus 的品牌色、功能色、中性色
- **布局规范**: 24 栅格系统,8px 基础间距
- **字体排版**: 14px 基准字号,1.5-1.8 行高
- **阴影规范**: 三层阴影系统

### 组件使用
- **按钮**: 主按钮在右,次按钮在左
- **表单**: 单列布局,即时校验
- **表格**: 信息层次清晰,操作位置合理
- **对话框**: 合理尺寸,清晰操作

### 响应式设计
- **断点**: xs/sm/md/lg/xl/xxl
- **移动优先**: 先设计移动端,再扩展到桌面端
- **灵活布局**: 使用相对单位和弹性布局

## 🚀 技术栈

- **框架**: Vue 3 + TypeScript
- **UI 库**: Element Plus
- **路由**: Vue Router 4
- **构建工具**: Vite
- **代码风格**: ESLint + Prettier

## 📦 运行项目

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```
